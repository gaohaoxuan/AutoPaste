# 剪贴板模块开发规范（Codex）

## 项目目标

开发一个运行于 **macOS** 的桌面剪贴板管理器。

开发环境：

* macOS
* 当前项目技术栈

运行环境：

* macOS

---

## 核心原则

本模块**仅参考 Windows 11 自带剪贴板（Win + V）的功能设计**。

包括：

* 功能逻辑

* 操作流程

* 用户交互

* 数据管理

* 历史记录机制

* 固定（Pin）

* 删除

* 清空

* 搜索

* 历史恢复

* 剪贴板监听

**不要模仿 Windows 的视觉风格。**

---

## UI 设计原则

界面应完全遵循 **macOS 的设计语言**，保持与整个应用一致。

包括但不限于：

* 使用 macOS 原生窗口风格

* 使用 macOS 常见的按钮和控件

* 使用符合 macOS 的间距和布局

* 使用系统字体（SF Pro / San Francisco，或项目统一字体）

* 支持浅色/深色模式

* 使用符合 macOS 的动画和交互

* 保持与现有应用整体风格一致

不要为了模仿 Windows 而复制其外观。

---

## 功能要求

功能应尽可能与 Windows 11 原生剪贴板保持一致。

至少包括：

### 剪贴板监听

自动监听系统剪贴板。

支持：

* 文本

* 图片

* HTML

* 文件

自动加入历史记录。

避免重复记录连续相同内容。

---

### 剪贴板历史

维护历史列表。

默认：

* 最大保存 100 条（可配置）。

超过限制：

自动删除最旧记录。

每条记录包含：

* 唯一 ID

* 内容类型

* 内容

* 预览

* 创建时间

* 是否固定

---

### 恢复历史

点击任意历史项：

立即恢复到系统剪贴板。

默认不自动粘贴，仅恢复剪贴板内容。

---

### 固定（Pin）

支持固定。

固定内容：

* 始终排在顶部

* 不参与自动清理

* 可取消固定

---

### 删除

支持：

* 删除单条

* 清空历史

清空历史时：

保留固定项。

---

### 搜索

支持搜索：

* 文本

* 文件名称

搜索结果实时更新。

---

### 图片

支持：

* PNG

* JPEG

* GIF

* WEBP（若系统支持）

* TIFF（macOS 常见格式）

显示缩略图。

---

### 文件

支持：

* 单文件

* 多文件

* 文件夹

恢复后可正常粘贴。

---

### HTML

保存 HTML 内容。

列表中显示纯文本预览。

恢复时保持原始 HTML 数据。

---

## 快捷键

默认快捷键：

**⌘ + Shift + V**

要求：

* 可修改

* 统一配置管理

* 不允许硬编码

---

## 数据持久化

程序退出时：

自动保存历史。

重新启动：

恢复历史。

建议：

* SQLite 保存元数据

* 图片保存到缓存目录

* 文件保存引用信息

---

## 架构要求

遵循当前项目架构。

建议划分：

clipboard/

    ClipboardManager

    ClipboardListener

    ClipboardHistory

    ClipboardStorage

    ClipboardSearch

    ClipboardHotkey

    ClipboardWindow

    ClipboardPreview

    ClipboardSettings

要求：

* 单一职责

* 高内聚、低耦合

* 易维护

* 易扩展

* UI 与业务逻辑分离

---

## 开发要求

开发过程中请遵循以下原则：

* 不修改已有功能

* 不影响现有模块

* 保持项目始终可编译

* 所有新增代码遵循当前代码规范

* 完整处理异常情况

* 所有配置集中管理

* 为未来扩展（如云同步、OCR、收藏等）预留接口，但本阶段不要实现

---

## 最终目标

实现一个**运行于 macOS、外观和交互符合 macOS 应用习惯**的剪贴板管理器，其**核心功能、行为逻辑和用户体验参考 Windows 11 的 Win + V 剪贴板**。在满足 macOS 平台设计规范的前提下，尽可能复刻 Windows 剪贴板的功能能力，而不是复刻其视觉风格。这样既能让用户获得成熟、高效的剪贴板管理体验，又能保持整个应用在 macOS 上的统一性和原生感。
